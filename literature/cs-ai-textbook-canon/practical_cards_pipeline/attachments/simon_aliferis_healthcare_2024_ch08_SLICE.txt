## scope

Chapter 8 (text lines **39738–40901**) is a **clinical/biomedical NLP primer** covering: corpus and token terminology; preprocessing (cleaning, stopwords, tokenization, stemming/lemmatization); feature extraction (n-grams, TF-IDF, embeddings via embedding layers, Word2Vec, GloVe); biomedical resources (UMLS, SemMedDB); clinical NLP tasks (NER, concept normalization, relation extraction, text classification, NLG, QA); symbolic tools (MetaMap, SemRep, cTAKES); ML paradigms (supervised, weakly supervised, unsupervised embeddings); deep learning (RNN/LSTM/biLSTM, transformers, BERT and biomedical variants); and graph-based NLP (GNNs, link prediction, edge classification, graph embeddings). Emphasis on starting simple, documenting experiments, and comparing against baselines before scaling to transformers.

Out of scope: full LLM safety/regulatory treatment (forward-referenced elsewhere in volume).

## key_findings

### Background and preprocessing

1. **Clinical corpora** — EHR notes, radiology/pathology reports, PubMed text; documents vs words vs tokens; preprocessing pipeline choice is task-dependent. [verified, 39777–39810]

2. **Cleaning and stopwords** — Character-set normalization, stopword removal justified by low semantic value despite Zipfian frequency (unlike typical ML "keep common signals"). [verified, 39812–39837]

3. **Tokenization** — Whitespace vs subword (WordPiece) for OoV mitigation. [verified, 39839–39853]

4. **Stemming vs lemmatization** — Stemming is fast but error-prone on irregular forms; lemmatization uses vocabulary/morphology. [verified, 39855–39877]

5. **BP 14.1** — Test multiple preprocessing pipelines for each goal. [verified, 39957–39960]

### Features and resources

6. **N-grams** — Word/token/character levels trade morphology vs semantics. [verified, 39888–39914]

7. **TF-IDF** — Weights token importance within document vs corpus. [verified, 39916–39921]

8. **Embeddings** — Embedding layers (task-specific, data-hungry), Word2Vec (CBOW/skip-gram), GloVe (global co-occurrence); reduce sparsity for ML. [verified, 39923–39955]

9. **UMLS** — Metathesaurus, Semantic Network, SPECIALIST Lexicon unify synonymous/hierarchical clinical concepts (CUIs). [verified, 39970–39988]

10. **SemMedDB** — >96M subject-predicate-object predications from SemRep over PubMed; SQL/graph-queryable. [verified, 39990–40013]

### Clinical NLP tasks

11. **Information extraction core** — NER (BIO labeling), concept normalization (lay vs clinical terms), RelEx (requires NER first); text classification at document level; NLG for synthetic notes and answers; QA = NLU + retrieval + generation. [verified, 40015–40157]

12. **Symbolic stack** — MetaMap (NER + normalization via UMLS); SemRep (rule-based RelEx); cTAKES (clinical pipeline including negation, PoS, etc.). [verified, 40159–40211]

13. **Pitfall 14.1** — Do not reimplement preprocessing when spaCy/cTAKES exist. [verified, 40213–40217]

### Machine learning for NLP

14. **Sparse high-dimensional inputs** — 15k unigram bag-of-words impractical without TF-IDF/embeddings. [verified, 40219–40235]

15. **BP 14.2 / Pitfall 14.2** — Document design rationale and log all experiments. [verified, 40237–40246]

16. **Supervised / weakly supervised** — BIO-labeled NER; heuristic/dictionary labels for supplements when gold labels absent. [verified, 40248–40272]

17. **Unsupervised embeddings** — Semantic similarity geometry; subword embeddings reduce OoV rigidity. [verified, 40274–40307]

18. **BP 14.3** — Grid search + k-fold CV for hyperparameters. [verified, 40309–40314]

### Deep learning and graphs

19. **DL rationale** — Language complexity favors high-capacity nonlinear models; capacity vs architectural innovation trends. [verified, 40316–40354]

20. **BP 14.4** — Permutation tests and covariance analysis before deployment. [verified, 40356–40360]

21. **RNN → LSTM → biLSTM** — Sequence modeling; vanishing gradients; bidirectional context preferred for language. [verified, 40362–40390]

22. **Transformers and BERT** — Self-attention encoder-decoder; masked LM pre-training + fine-tuning; biomedical BERT family (Bio-BERT, BioClinical-BERT, PubMed-BERT, Blue-BERT) on Hugging Face. [verified, 40392–40427]

23. **BP 14.5 / Pitfall 14.3** — Always baseline (rules, linear models, vanilla BERT); avoid jumping to SOTA repos without proof of need. [verified, 40429–40445]

24. **Graph NLP** — Nodes (drugs, diseases, proteins), edges (treats, affects); GNN message passing; link prediction for drug repurposing; edge classification for interaction types; Euclidean/hyperbolic graph embeddings. [verified, 40447–40504]
