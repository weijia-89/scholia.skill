## scope

**Part V — Maintainability** opens with testing strategy: scope (unit/integration/E2E pyramid), size (small/intermediate/large), test doubles, contract tests, risk-based E2E (GDPR purge), and **TLA+ formal verification** with dual-write migration example.

---

## key_findings

### Value & limits

- Late bug detection is expensive; tests enable safe change (8814–8822).
- Can't predict all distributed states; best-effort only (8824–8830).
- **Rule:** if you need confidence in behavior, write a test (8841–8843).

### Scope (§29.1)

| Type | Scope | Notes |
|------|-------|-------|
| Unit | Small (class) | Public API; state not sequence; sociable style (Fowler) |
| Integration (narrow) | Service + one dependency | Fowler narrow |
| E2E | Multi-service user scenarios | Slow, flaky; user-journey framing reduces count (8900–8905) |
| Pyramid | Many unit, fewer integration, fewest E2E | Fig 29.1 (8911–8914) |

### Size (§29.2)

- **Small:** single process, no I/O — fast, deterministic (8935–8937).
- **Intermediate:** single node, local I/O (8939–8942).
- **Large:** multi-node (8944–8945).
- Doubles: fake > stub > mock (brittle last resort) (8949–8974).
- **Contract tests** for integration with mocks (8976–8983).

### Practical considerations (§29.3)

- Smallest test with real fakes where possible (8997–9018).
- **GDPR user purge:** high-risk → periodic **production E2E** with live services (9020–9032).

### Formal verification (§29.4)

- Spec before code catches design bugs (9036–9057).
- **TLA+:** behaviors as state sequences; safety (invariant) vs liveness (9036–9087).
- **Dual-write migration** A→B: model checker finds crash-between-writes liveness violation; concurrent writers ordering issue (9089–9126).
- Fix via serialized write channel (global order) (9127–9132).
