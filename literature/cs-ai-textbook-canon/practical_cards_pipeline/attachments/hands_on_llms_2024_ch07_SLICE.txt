## scope

Chapter 7 extends Ch. 6 prompting with **LangChain-orchestrated LLM systems** (lines 7254–8326):

1. **Model I/O** — GGUF quantized Phi-3 via `LlamaCpp`; quantization intuition (≥4-bit rule of thumb); `invoke` requires explicit Phi-3 template.

2. **Chains** — `prompt | llm` pipe syntax; reusable `PromptTemplate`; sequential `LLMChain` story pipeline (title → character → story).

3. **Memory** — Stateless LLM problem; `ConversationBufferMemory`, `ConversationBufferWindowMemory` (k=2), `ConversationSummaryMemory` with secondary summarization LLM; Table 7-1 trade-offs.

4. **Agents** — ReAct loop (Thought / Action / Observation); tool inventory (DuckDuckGo search, `llm-math` calculator); `create_react_agent` + `AgentExecutor`; GPT-3.5 required for reliable agent demo (Phi-3 insufficient).

5. **Framework context** — LangChain as primary; DSPy and Haystack noted; retrieval deferred to Ch. 8.

6. **Chapter close** — Agents + tools foundation for semantic search / RAG (Ch. 8). [verified from text, lines 7254–8326]

## key_findings

1. **No fine-tuning required** — Chapter continues inference-time techniques (prompting, orchestration) before Ch. 12 training. [verified, 7256–7272]

2. **LangChain composition model** — Model I/O, memory, agents, chains as modular blocks; combined performance exceeds isolated use. [verified, 7274–7304]

3. **Quantization for local inference** — GGUF compresses Phi-3; FP16 example; 4-bit minimum recommended; 2–3 bit degrades vs smaller higher-precision model. [verified, 7311–7351]

4. **Template gap in raw invoke** — `llm.invoke` without template returns empty; chains attach Phi-3 `<s><|user|>…<|assistant|>` template. [verified, 7378–7504]

5. **Sequential chains** — `title | character | story` LLMChains pass outputs as inputs; decomposes complex generation (Figure 7-7/7-8). [verified, 7542–7667]

6. **Memory = prompt injection** — Buffer memory appends full `chat_history` to prompt; windowed buffer drops older turns beyond k. [verified, 7673–7881]

7. **Summary memory trade-off** — Extra LLM call per turn summarizes history; compresses tokens but may lose explicit facts (age forgotten while name inferred). [verified, 7888–8018]

8. **Memory comparison (Table 7-1)** — Buffer: no loss within window but token-heavy; window: bounded but drops early context; summary: long conversations possible but summarization quality + latency. [verified, 8020–8062]

9. **Agents vs fixed chains** — Agents let LLM choose actions/tools/order; ReAct merges reasoning and tool use (Yao et al. 2022). [verified, 8064–8157]

10. **ReAct template** — Question → Thought → Action → Action Input → Observation loop until Final Answer; MacBook USD→EUR example with search + calculator. [verified, 8136–8287]

11. **Agent reliability warning** — Autonomous intermediate steps lack human-in-the-loop; answers may be wrong despite impressive traces; suggest URL logging or step confirmation. [verified, 8292–8300]

12. **Compute constraint** — Small local Phi-3 inadequate for ReAct; GPT-3.5 used; field may catch up. [verified, 8166–8191]
