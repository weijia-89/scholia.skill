## key_findings

### Runtime configuration

- Settings (behavior + secrets) must not be hardcoded; external store (AWS AppConfig, Azure App Configuration) (10307–10315).
- Deploy-time env vars: can't change without redeploy (10317–10332).
- **Periodic re-read** at runtime; recreate dependents when settings change (e.g. HTTP handler) (10333–10337).

### Feature flags & A/B

- Toggle features enable/disable per instance or user fraction (10338–10343).
- Release build with feature off → gradual enable; same mechanism for **A/B tests** (10344–10350).

### Part V Summary (in slice)

- Most engineering time is maintenance, not greenfield design (10353–10359).
- "Sixth sense" from operating production beats trend-chasing (10360–10371).
- Advice: on-call, postmortems, ask how incidents were avoidable (10366–10371).
