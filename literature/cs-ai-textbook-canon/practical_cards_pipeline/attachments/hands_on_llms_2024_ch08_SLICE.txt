## scope

Chapter 8 is the **search + RAG procedural spine** (lines 8327–9486):

1. **Industry context** — BERT in Google Search / Bing; semantic search vs keyword; hallucinations → RAG (Lewis et al. 2020).

2. **Three categories** — Dense retrieval (embedding k-NN), reranking (cross-encoder relevance scores), RAG (retrieve + grounded generation with citations).

3. **Dense retrieval walkthrough** — Cohere embed Interstellar Wikipedia sentences; FAISS `IndexFlatL2`; semantic vs BM25 keyword search; similarity thresholds; hybrid search recommendation; domain shift caveat; chunking strategies (one vs multi-vector per doc, overlap, title context).

4. **Scale** — NumPy for small indexes; Annoy/FAISS ANN at millions; vector DBs (Weaviate, Pinecone) for incremental updates/filters; retrieval fine-tuning with pos/neg query-doc pairs (Ch. 10 depth).

5. **Reranking** — Cohere Rerank API; BM25→rerank pipeline; monoBERT cross-encoder; MIRACL nDCG@10 boost 36.5→62.8 cited; Sentence Transformers local path.

6. **Evaluation** — MAP / average precision / mean average precision; nDCG for graded relevance; test suite = corpus + queries + judgments.

7. **RAG implementations** — Cohere `co.chat` with documents + span citations; local LangChain `RetrievalQA` + FAISS + `thenlper/gte-small` + Phi-3 GGUF prompt template.

8. **Advanced RAG** — Query rewriting, multi-query, multi-hop, query routing, agentic RAG; Ragas + LLM-as-judge metrics (faithfulness, answer relevance, citation precision/recall per Liu et al. 2023).

## key_findings

1. **Semantic search = embedding proximity** — Query and documents embedded to same space; nearest neighbors are results; may need similarity threshold when corpus lacks answers. [verified, 8414–8449, 8664–8686]

2. **Dense beats lexical on meaning** — "how precise was the science" retrieves accuracy sentence without shared keywords; BM25 favors "science fiction" opener. [verified, 8583–8657]

3. **Chunking is a design parameter** — Transformer context limits require splits; multi-vector per document preferred over single averaged vector; sentence vs paragraph vs overlapping chunks (Figures 8-7–8-10). [verified, 8707–8794]

4. **FAISS baseline** — `IndexFlatL2` exact search fine for small corpora; `input_type` distinction `search_document` vs `search_query` in Cohere API. [verified, 8528–8578]

5. **Hybrid search advised** — Dense retrieval weak on exact phrase match; combine with keyword (BM25). [verified, 8688–8692]

6. **Domain generalization limit** — Retrieval models trained on web/Wikipedia degrade on out-of-domain legal text without domain training data. [verified, 8694–8698]

7. **Reranker role** — Second-stage cross-encoder scores query+doc jointly; shortlist 100–1000 from stage-1 then rerank top-k; monoBERT = classification 0–1 relevance. [verified, 8863–9007]

8. **BM25 + rerank demo** — Reranker elevates scientifically relevant sentence above lexical false positive. [verified, 8925–8974]

9. **MAP intuition** — Rewards ranking relevant docs higher; mean over queries → MAP for system comparison; nDCG handles graded relevance. [verified, 9013–9128]

10. **RAG = search + grounded generation** — Retrieve top docs → prompt LLM with context; enables chat-with-data, reduces hallucination (not eliminates). [verified, 9130–9168]

11. **Managed RAG with citations** — Cohere chat returns `ChatCitation` spans tied to `document_ids`; local Phi-3 RAG loses span citations. [verified, 9178–9311]

12. **LangChain RAG stack** — `HuggingFaceEmbeddings` + `FAISS.from_texts` + custom Phi-3 template + `RetrievalQA` chain_type `stuff`. [verified, 9226–9311]

13. **Advanced patterns** — Query rewrite for verbose chat; multi-query for comparisons; multi-hop sequential searches; route by datasource (HR vs CRM); agentic RAG delegates planning to capable models (Command R+ cited). [verified, 9316–9409]

14. **RAG eval axes** — Fluency, perceived utility, citation recall/precision (Liu et al. 2023); Ragas automates faithfulness and answer relevance. [verified, 9411–9447]
