## key_findings

1. **Open-source default** — Book favors open models for flexibility; start small (Phi-3 3.8B, ≤8 GB VRAM) then scale up. [verified, 6064–6088]

2. **Chat templates are contractual** — `apply_chat_template` reveals training-time format; pipeline hides this but custom templates needed elsewhere (LangChain Ch. 7). [verified, 6137–6156]

3. **Deterministic vs creative generation** — `do_sample=False` → greedy most-likely token; `do_sample=True` enables temperature/top_p stochasticity. [verified, 6189–6224]

4. **Temperature vs top_p** — Temperature reshapes token probability distribution; top_p truncates nucleus by cumulative probability; Table 6-1 pairs them per task (e.g. high temp + low top_p for coherent creativity). [verified, 6195–6268]

5. **Prompt = iterative experiment** — No perfect prompt; components composable; model-specific effectiveness varies. [verified, 6278–6282, 6507–6511]

6. **Instruction prompting tactics** — Specificity reduces drift; hallucination guard via abstention instruction; place critical info at start or end (middle forgotten — Liu et al. 2023). [verified, 6373–6397]

7. **Modular prompt components** — Persona, instruction, context, format, audience, tone, data — add/remove/order iteratively (Figure 6-11/6-12 walkthrough with paper summarization). [verified, 6420–6496]

8. **In-context learning** — Zero/one/few-shot; exemplar "Gigamuru"/"screeg" one-shot via user/assistant turns in chat template. [verified, 6513–6586]

9. **Chain prompting** — Sequential prompts: product name+slogan → sales pitch; enables per-step token budgets and validation passes. [verified, 6593–6672]

10. **Reasoning is mimicked, not guaranteed** — LLM "reasoning" attributed to pattern matching/memorization; prompt engineering emulates System 2 slow thinking. [verified, 6688–6698]

11. **Chain-of-thought** — Few-shot reasoning exemplars improve math; zero-shot trigger "Let's think step-by-step" (Kojima et al.); variant phrases exist. [verified, 6718–6803]

12. **Self-consistency** — Sample multiple CoT paths; majority vote on final answer; n× latency cost. [verified, 6805–6829]

13. **Tree-of-thought** — Explore/prune intermediate thoughts; zero-shot multi-expert prompt emulates multi-call ToT in one generation. [verified, 6831–6898]

14. **Output control triad** — Examples (few-shot structure), grammar (constrained decoding), fine-tuning (Ch. 12); production needs structured/valid/ethical/accurate outputs. [verified, 6900–6953]

15. **JSON via grammar** — `llama_cpp` `response_format={"type": "json_object"}` enforces valid JSON at sampling time vs unreliable zero-shot JSON. [verified, 7079–7193]
